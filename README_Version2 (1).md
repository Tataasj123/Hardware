# เข็มขัดตรวจวัดการหกล้ม (Fall Detection Belt System)

ระบบ IoT แบบสองบอร์ดออกแบบมาเพื่อตรวจจับการหกล้ม แจ้งเตือนผู้ดูแล และติดตามตำแหน่งผู้สูงอายุและผู้พิการแบบเรียลไทม์

---

## 📋 ภาพรวมโปรเจกต์

ระบบนี้ประกอบด้วยสองส่วนหลัก:
- **บอร์ดที่ 1**: บอร์ดเข็มขัดของผู้สูงอายุ - ตรวจจับการหกล้มและส่งการแจ้งเตือน
- **บอร์ดที่ 2**: บอร์ดติดตามสำหรับผู้ดูแล - รับการแจ้งเตือนการหกล้มพร้อมระดับความรุนแรง
- **การอินทิเกรต Google Chat**: ส่งการแจ้งเตือนและข้อมูลตำแหน่งไปยังสมาร์ทโฟนผ��้ดูแล

### คุณสมบัติหลัก
✅ ตรวจจับการหกล้มแบบเรียลไทม์ด้วย Accelerometer (MPU6050)  
✅ จำแนกระดับความรุนแรงตามการเปลี่ยนแปลงของค่าความเร่ง  
✅ ติดตามตำแหน่งผ่าน GPS (GY-NEO-6M)  
✅ แจ้งเตือนเสียง (Buzzer) และ ไฟ (LED) ทันที  
✅ อินทิเกรต Google Chat สำหรับแจ้งเตือนผ่านสมาร์ทโฟน  
✅ จัดการการแจ้งเตือนเท็จด้วยสวิตช์ manual override  
✅ หน้าจอ OLED แสดงระดับความรุนแรงของการหกล้ม  

---

## 🏗️ สถาปัตยกรรมระบบ

### บอร์ดที่ 1: บอร์ดเข็มขัดของผู้สูงอายุ
**หน้าที่**: ตรวจจับการหกล้มและสร้างการแจ้งเตือนครั้งแรก

**อุปกรณ์**:
- ไมโครคอนโทรลเลอร์ ESP32
- Accelerometer MPU6050 (ตรวจจับการเปลี่ยนแปลงความเร่งแนวแกน X, Y, Z)
- โมดูล GPS GY-NEO-6M (ติดตามตำแหน่ง)
- Active Buzzer
- ไฟ LED สีแดง (ตัวบ่งชี้)
- ปุ่มสวิตช์ (ยกเลิกการแจ้งเตือน)
- Breadboard และสายเดินไฟ

**ขั้นตอนการทำงาน**:
1. MPU6050 ตรวจสอบความเร่งแนวแกน X, Y, Z อย่างต่อเนื่อง
2. เมื่อตรวจพบการเปลี่ยนแปลงความเร่งฉับพลัน → จำแนกระดับความรุนแรงของการหกล้ม
3. ส่งข้อมูลการหกล้ม + พิกัด GPS ไปยังบอร์ดที่ 2
4. ส่งการแจ้งเตือนไปยัง Google Chat พร้อมกับ:
   - ตำแหน่ง (พิกัด Google Maps)
   - ระดับความรุนแรงของการหกล้ม
5. เปิดใช้ Buzzer และ LED สีแดง
6. ผู้สูงอายุสามารถยกเลิกการแจ้งเตือนได้โดยกดปุ่มสวิตช์

**ระดับความรุนแรงของการหกล้ม**: กำหนดจากช่วงค่าความเร่ง
- ระดับต่ำ
- ระดับกลาง
- ระดับสูง

---

### บอร์ดที่ 2: บอร์ดติดตามของผู้ดูแล
**หน้าที่**: รับและแสดงการแจ้งเตือนการหกล้มพร้อมป้อนกลับเสียง/ภาพที่เหมาะสม

**อุปกรณ์**:
- ไมโครคอนโทรลเลอร์ ESP32
- หน้าจอ OLED (แสดงระดับความรุนแรงของการหกล้ม)
- Active Buzzer (เสียงแจ้งเตือนต่างกันตามระดับความรุนแรง)
- ไฟ LED สีแดง (ตัวบ่งชี้)
- ปุ่มสวิตช์ (หยุด Buzzer/LED)
- Breadboard และสายเดินไฟ

**ขั้นตอนการทำงาน**:
1. รับข้อมูลการหกล้มจากบอร์ดที่ 1
2. Buzzer ส่งเสียงพร้อมรูปแบบเสียงต่างกันตามความรุนแรง:
   - ระดับต่ำ: บีปเดี่ยว
   - ระดับกลาง: บีปคู่
   - ระดับสูง: บีปต่อเนื่อง
3. LED สีแดงกระพริบทันที
4. หน้าจอ OLED แสดงระดับความรุนแรง
5. ผู้ดูแลสามารถหยุด Buzzer/LED ได้โดยกดปุ่มสวิตช์

---

### การอินทิเกรต Google Chat
- ส่งการแจ้งเตือนผ่าน Push Notification ไปยังสมาร์ทโฟนผู้ดูแล
- ข้อมูลที่ส่ง:
  - ตำแหน่ง GPS แบบเรียลไทม์ของผู้สูงอายุ
  - จำแนกระดับความรุนแรงของการหกล้ม
  - เวลาที่เกิดเหตุการณ์
- ช่วยให้ผู้ดูแลสามารถติดตามได้แม้ไม่อยู่ใกล้บอร์ดที่ 2

## 🛠️ รายการอุปกรณ์

|--------|-------|----------|
| ESP32 | 2 | ไมโครคอนโทรลเลอร์สำหรับทั้งสองบอร์ด |
| MPU6050 | 1 | Accelerometer (ตรวจจับการหกล้ม) |
| GY-NEO-6M | 1 | โมดูล GPS (ติดตามตำแหน่ง) |
| หน้าจอ OLED | 1 | แสดงระดับความรุนแรง (บอร์ดที่ 2) |
| Active Buzzer | 2 | แจ้งเตือนเสียง (หนึ่งตัวต่อบอร์ด) |
| ไฟ LED สีแดง | 2 | ตัวบ่งชี้ภาพ (หนึ่งตัวต่อบอร์ด) |
| ปุ่มสวิตช์ | 2 | ยกเลิก/หยุด (หนึ่งตัวต่อบอร์ด) |
| Breadboard | 2 | ประกอบวงจร |
| สายจัมเปอร์ | หลายเส้น | การเชื่อมต่อ |
| แหล่งจ่ายไฟ | 2 | 5V USB หรือแบตเตอรี่ |

## 📡 โปรโตคอลการสื่อสาร

**บอร์ดที่ 1 → บอร์ดที่ 2**: การสื่อสารแบบไร้สาย (WiFi/BLE ผ่าน ESP32)
- รูปแบบข้อมูล: `{fall_severity, gps_lat, gps_lon, timestamp}`

**บอร์ดที่ 1 → Google Chat**: เรียก HTTPS API
- ส่งข้อความแจ้งเตือนพร้อมลิงก์ตำแหน่งและระดับความรุนแรง

## 🔧 การติดตั้งและตั้งค่า

### ความต้องการก่อนการตั้งค่า
- Arduino IDE หรือ PlatformIO
- ไลบรารี่:
  - `MPU6050` (accelerometer)
  - `TinyGPS++` (GPS parsing)
  - `Adafruit_SSD1306` (OLED display)
  - `HTTPClient` (สำหรับ Google Chat API)
  - WiFi library (built-in ESP32)

### ตั้งค่าบอร์ดที่ 1 (เข็มขัดของผู้สูงอายุ)
1. ติดตั้งโมดูล MPU6050 และ GPS
2. เชื่อมต่อ Buzzer และ LED ไปยัง GPIO pins
3. กำหนด URL webhook ของ Google Chat
4. อัปโหลด Firmware พร้อมอัลกอริทึมตรวจจับการหกล้ม
5. ปรับแต่งค่าเกณฑ์ความเร่งสำหรับตรวจจับการหกล้ม

### ตั้งค่าบอร์ดที่ 2 (จอตรวจสอบของผู้ดูแล)
1. ติดตั้งหน้าจอ OLED
2. เชื่อมต่อ Buzzer และ LED ไปยัง GPIO pins
3. กำหนด WiFi สำหรับรับข้อมูลจากบอร์ดที่ 1
4. อัปโหลด Firmware สำหรับติดตาม
5. ทดสอบการรับการแจ้งเตือน

## ⚙️ คุณสมบัติซอฟต์แวร์

### อัลกอริทึมการตรวจจับการหกล้ม
```
หากค่าความเร่งเปลี่ยนแปลง > ค่าเกณฑ์
  จำแนกความรุนแรงตามขนาดความเร่ง
  ส่งแจ้งเตือนไปยังบอร์ดที่ 2
  เปิดใช้ Buzzer & LED
  ได้รับตำแหน่ง GPS
  ส่งไปยัง Google Chat
```

### การจัดการการแจ้งเตือนเท็จ
- หากเซนเซอร์ตรวจจับการหกล้มแต่ข้อมูลไม่ถูกต้อง (เช่น อุปกรณ์ตกพื้น)
- Buzzer และ LED ยังคงเปิดใช้งาน
- ผู้สูงอายุสามารถยกเลิกได้ด้วยปุ่มสวิตช์
- ป้องกันการแจ้งเตือนผู้ดูแลที่ไม่จำเป็น

### จำแนกระดับความรุนแรง
- **ระดับต่ำ**: การเปลี่ยนแปลงความเร่ง 20-40 m/s²
- **ระดับกลาง**: การเปลี่ยนแปลงความเร่ง 40-70 m/s²
- **ระดับสูง**: การเปลี่ยนแปลงความเร่ง >70 m/s²

   ```
     "location": "https://maps.google.com/?q=13.7563,100.5018",
     "timestamp": "2026-03-12 14:30:45"
   }
   ```

## 🚀 วิธีการใช้งาน

### สำหรับผู้สูงอายุ (บอร์ดที่ 1)
1. สวมเข็มขัดพร้อมบอร์ดและเซนเซอร์
2. ระบบทำงานอย่างต่อเนื่องในพื้นหลัง
3. หากตรวจพบการหกล้ม:
   - Buzzer ส่งเสียง
   - LED สีแดงกระพริบ
   - ส่งตำแหน่งไปยังผู้ดูแล
4. กดปุ่มสวิตช์เพื่อยกเลิกการแจ้งเตือนเท็จ

### สำหรับผู้ดูแล (บอร์ดที่ 2)
1. เก็บบอร์ดติดตามไว้ใกล้ ๆ
2. รับการแจ้งเตือนเสียง/ภาพเมื่อตรวจพบการหกล้ม
3. ดูหน้าจอ OLED สำหรับระดับความรุนแรง
4. ดูตำแหน่งจากการแจ้งเตือน Google Chat
5. กดปุ่มสวิตช์เพื่อหยุด Buzzer/LED

## ⚠️ ข้อควรระวัง

- ทดสอบอัลกอริทึมการตรวจจับการหกล้มก่อนการใช้งานจริง

- ใช้แหล่งจ่ายไฟที่เнадежный พร้อมแบตเตอรี่สำรอง
- ตรวจสอบการปรับเทียบเซนเซอร์อย่างสม่ำเสมอ
- ตรวจสอบว่าการแจ้งเตือน Google Chat ทำงานได้
- ทดสอบระยะการสื่อสารระหว่างบอร์ด
- เก็บหมายเลขโทรศัพท์ติดต่อฉุกเฉินไว้
